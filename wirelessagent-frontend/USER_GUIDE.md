# 5G网络切片资源分配系统 - 使用说明

## 系统概述

本系统为5G网络切片资源分配提供了一个可视化的前端界面，支持上传CSV格式的用户数据文件，执行网络切片资源分配，并实时显示处理过程和最终分配结果。

## 快速开始

### 方法一：使用启动脚本（推荐）

1. **安装前端依赖**
   ```bash
   双击运行 install_frontend.bat
   ```

2. **启动系统**
   ```bash
   双击运行 start.bat
   ```

3. **打开浏览器**
   - 前端界面：http://localhost:5173
   - 后端API：http://localhost:8000
   - API文档：http://localhost:8000/docs

### 方法二：手动启动

#### 启动后端服务器

```bash
cd F:\code\wirelessAgent_backend
python api_server.py
```

#### 启动前端开发服务器

```bash
cd F:\code\wirelessAgent_backend\wirelessagent-frontend
npm run dev
```

## 界面功能

### 1. 文件上传区域（左侧面板）

- **拖拽上传**：将CSV文件拖拽到上传区域
- **点击上传**：点击上传区域选择文件
- **文件信息**：显示文件名和文件大小
- **开始处理**：点击按钮开始处理上传的文件
- **清除文件**：清除已上传的文件

**支持的文件格式**：.csv

**CSV文件格式要求**：
```
RX_ID,X,Y,Z,SNR_dB,RX_Power_dBm,CQI,LOS,User_Request,Request_Label
1,75.64,182.46,1.50,6.23,-86.76,7,0,I need to stream 4K video,eMBB
2,-301.92,20.62,1.50,7.63,-85.36,7,0,I want to watch HD video,eMBB
```

### 2. 处理日志区域（中间面板）

实时显示处理过程中的日志信息：

- **蓝色**：信息日志（info）
- **绿色**：成功日志（success）
- **橙色**：警告日志（warning）
- **红色**：错误日志（error）

**功能按钮**：
- **清空日志**：清除所有日志记录
- **导出日志**：将日志导出为文本文件

### 3. 分配结果区域（右侧面板）

#### 详细数据标签页

以表格形式显示每个用户的分配结果：

| 列名 | 说明 |
|------|------|
| 用户ID | 用户标识符 |
| 用户请求 | 用户的网络需求描述 |
| CQI | 信道质量指示器（1-15） |
| 切片类型 | 分配的切片类型（eMBB/URLLC/mMTC） |
| 带宽(MHz) | 分配的带宽 |
| 速率(Mbps) | 计算的数据传输速率 |
| 延迟(ms) | 分配的延迟 |
| 状态 | 分配状态（成功/失败） |

#### 统计概览标签页

以卡片形式显示关键统计指标：

- **成功分配**：成功分配的用户数量
- **分配失败**：分配失败的用户数量
- **成功率**：分配成功的百分比
- **eMBB切片**：分配到eMBB切片的用户数
- **URLLC切片**：分配到URLLC切片的用户数
- **mMTC切片**：分配到mMTC切片的用户数
- **平均速率**：所有成功分配用户的平均速率
- **动态调整次数**：执行动态资源调整的次数

**功能按钮**：
- **导出结果**：将分配结果导出为CSV文件
- **清空结果**：清除所有分配结果

## 处理流程

1. **上传CSV文件**
   - 将包含用户数据的CSV文件上传到系统

2. **文件解析**
   - 系统自动解析CSV文件，提取用户信息

3. **网络切片分配**
   - 对每个用户执行以下步骤：
     - 意图理解：分析用户请求，确定网络需求
     - 切片选择：根据意图选择合适的切片类型
     - 资源分配：计算带宽、速率和延迟
     - 容量检查：验证切片容量，必要时动态调整

4. **结果展示**
   - 在结果面板显示所有用户的分配结果
   - 提供统计概览和详细数据

5. **数据导出**
   - 可将结果导出为CSV文件进行进一步分析

## 切片类型说明

### eMBB（增强移动宽带）
- **适用场景**：高带宽应用
- **带宽范围**：6-20 MHz
- **速率范围**：100-400 Mbps
- **延迟范围**：10-100 ms
- **典型应用**：视频流、大文件下载、AR/VR、高清视频会议

### URLLC（超可靠低延迟通信）
- **适用场景**：低延迟应用
- **带宽范围**：1-5 MHz
- **速率范围**：1-100 Mbps
- **延迟范围**：1-10 ms
- **典型应用**：远程控制、自动驾驶、工业自动化、实时监控

### mMTC（大规模机器类型通信）
- **适用场景**：大规模物联网设备
- **带宽范围**：1-3 MHz
- **速率范围**：0.1-1 Mbps
- **延迟范围**：100-1000 ms
- **典型应用**：智能电表、环境监测传感器、资产追踪

## 常见问题

### Q1: 前端无法连接后端API？

**解决方案**：
1. 确认后端服务器正在运行（http://localhost:8000）
2. 检查API地址配置是否正确（src/services/api.ts）
3. 查看浏览器控制台的网络请求错误

### Q2: 文件上传失败？

**解决方案**：
1. 确认CSV文件格式正确
2. 检查文件大小是否过大
3. 查看处理日志中的错误信息

### Q3: 处理速度很慢？

**解决方案**：
1. 减少CSV文件中的用户数量
2. 检查系统资源使用情况（CPU、内存）
3. 关闭其他占用资源的程序

### Q4: 如何查看API文档？

访问 http://localhost:8000/docs 查看完整的API文档和测试接口。

### Q5: 如何修改界面样式？

- 全局样式：在 `src/App.vue` 的 `<style>` 部分修改
- 组件样式：在对应Vue组件的 `<style scoped>` 部分修改
